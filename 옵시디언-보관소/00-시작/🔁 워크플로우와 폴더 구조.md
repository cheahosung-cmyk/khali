# 🔁 워크플로우와 폴더 구조

## 기본 흐름: 수집 → 정리 → 연결 → 활용

```
📥 수집      → 떠오르는 것 일단 10-노트에 적기
🗂️ 정리      → 제목/태그 달고 폴더로 이동
🔗 연결      → [[링크]]로 관련 노트끼리 잇기
🚀 활용      → 글쓰기·프로젝트·복습에 사용
```

> [!tip] 핵심 원칙
> **완벽하게 분류하려 하지 마세요.** 일단 적고, 나중에 링크로 연결하는 게 옵시디언의 힘입니다.

---

## 추천 폴더 구조 (이 보관소)

| 폴더 | 용도 |
|---|---|
| `00-시작` | 가이드·규칙·이 보관소 사용법 |
| `10-노트` | 일상 메모, 아이디어, 일일 노트 |
| `20-템플릿` | 반복 양식 (일일/회의/독서 노트) |
| `30-자료` | 웹 스크랩, 참고 문헌, 인용 |
| `90-보관` | 끝난 노트 아카이브 |

> [!note] PARA 방식도 인기
> 폴더를 **Projects(프로젝트) / Areas(영역) / Resources(자료) / Archive(보관)** 4개로 나누는 방법입니다. 익숙해지면 시도해 보세요.

---

## 매일 하는 루틴 (예시)

1. 아침: [[📅 일일 노트 템플릿]]으로 오늘 노트 생성
2. 수시로: 떠오르는 생각 바로 적기 (`Ctrl/Cmd + N`)
3. 저녁: 오늘 노트를 관련 주제 노트와 `[[링크]]` 연결
4. 주말: `90-보관`으로 다 쓴 노트 정리
