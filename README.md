# khali — 한국 주식(KIS) 자동매매 시스템

> ⚠️ **면책**: 자동매매는 수익을 보장하지 않습니다. 이 프로그램이 보장하는 것은
> *수익*이 아니라 **규율(리스크 통제)·재현성·검증 가능성**입니다. 실거래 전에
> 반드시 백테스트 → 모의투자(페이퍼) → 소액 실거래 순서를 지키세요.
> 투자 손실의 책임은 전적으로 사용자에게 있습니다.

## 설계 철학

전문가 패널(퀀트 / 리스크 / 아키텍트 / 브로커API / 컴플라이언스) 토론으로 도출:

1. **브로커 중립 코어** — 전략·리스크 코드는 어느 브로커인지 모른다. 어댑터로 교체.
2. **동일 코드 검증** — 백테스트·페이퍼·실거래가 *같은* 전략/리스크 객체를 쓴다.
3. **리스크가 게이트키퍼** — 모든 주문은 `RiskManager`를 통과해야 한다.
4. **단계적 전개** — 백테스트에서 검증 안 된 전략은 실거래로 못 간다.

## 현재 채택 전략 (2026-06 시장 기준)

시장 진단: KOSPI ~8,900p, 반도체/AI 슈퍼사이클 **강추세 + 극단적 변동성(하루 ±10%)
+ 대형주 쏠림**. → 평균회귀는 부적합, **변동성 돌파 + 추세 필터 + ATR** 채택.

- **진입**: 당일 시가 + (전일 고저폭 × `k`) 돌파 시 매수
- **추세 필터**: 종가가 `N`일 이동평균 위일 때만 (하락반전 자동 회피)
- **리스크**: ATR 기반 손절 + 1%룰 사이징 + 종목 비중상한 + 일일 kill-switch
- **청산**: 당일 종가 청산 (오버나잇 갭 회피, 초기 버전)

## 구조

```
src/khali/
├─ models.py          # Bar, Order, Position, Account, Signal
├─ config.py          # KIS 자격증명 (.env에서만)
├─ data/feed.py       # CSV / 합성 시세 피드
├─ strategy/          # base + volatility_breakout (백테스트·실거래 공용)
├─ risk/manager.py    # ★포지션사이징·손절·일일한도·kill-switch
├─ broker/            # base / paper(모의체결) / kis(실거래 스켈레톤)
├─ engine/backtest.py # 봉 단위 구동 + 성과(수익률/MDD/승률)
└─ main.py            # CLI
```

## 빠른 시작

```bash
# 1) 백테스트 (외부 의존성 없이 합성 데이터로 즉시 실행)
PYTHONPATH=src python -m khali.main backtest --symbol 005930 --k 0.5 --ma 5

# 2) 페이퍼 실시간 루프 데모 (실데이터, 키 불필요)
PYTHONPATH=src python -m khali.main paper --ticks 15

# 3) 테스트
PYTHONPATH=src python -m pytest tests/ -v

# 4) 실거래 준비 (선택)
cp config/.env.example .env   # 키 입력 후
# KIS Developers에서 키 발급 → 모의투자(KIS_IS_PAPER=true)부터
# LiveSession의 PaperBroker를 KISBroker로 교체하면 모의/실거래 구동
```

추세 시스템의 본질: 강세장은 못 따라가도, **하락장에서 손실을 줄여** 장기
생존을 노린다. 단, 2026 데이터 일부는 합성이므로 **최종 검증은 KIS 실데이터로**.

## ⚠️ 안전 수칙

- API 키는 **절대** 코드/깃에 넣지 않는다 (`.env`만, `.gitignore` 등록됨).
- `KIS_IS_PAPER=false`(실전)는 백테스트·모의 충분히 검증 후에만.
- 실거래 어댑터(`broker/kis.py`)의 `submit`은 의도적으로 미구현 상태다.
