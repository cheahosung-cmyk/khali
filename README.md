# Khali — 빗썸 개인용 자동 코인매매 시스템

리플(XRP) 등 빗썸 마켓을 대상으로 하는 **개인용** 자동매매 시스템입니다.
수익률을 지키는 핵심은 리스크 관리라는 원칙 아래, 모든 전략 위에 강제
손절·익절·포지션 사이징·일일 손실 한도 레이어를 둡니다.

> ⚠️ **투자 위험 고지**: 자동매매는 실제 자금 손실 위험이 있습니다. 반드시
> `backtest → paper → live` 순서로 충분히 검증한 뒤, 잃어도 되는 소액으로만
> 시작하세요. 과거 성과가 미래 수익을 보장하지 않습니다.

## 안전 3단계

| 모드 | 설명 | 실제 돈 |
|------|------|:------:|
| `backtest` | 과거 캔들로 전략 수익률·MDD·승률 검증 | ❌ |
| `paper` | 실시간 시세 + 가상 주문 (기본값) | ❌ |
| `live` | 실제 빗썸 주문 실행 | ✅ |

`.env` 의 `ORDER_MODE` 한 곳만 바꿔 전환합니다. 기본값은 `paper` 이라
실수로 돈이 나가지 않습니다.

## 설치

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
cp .env.example .env   # 키/파라미터 편집
```

## 빗썸 API 키 (live 모드만 필요)

빗썸 **API 1.0**(Connect Key + HMAC-SHA512)과 **2.0**(JWT)을 모두 지원합니다.
`.env` 의 `API_VERSION` 으로 선택하며 기본값은 **1**(1.0)입니다.

1. 빗썸 → 마이페이지 → API 관리에서 키 발급
2. **반드시 '자산조회 + 거래' 권한만 부여하고 '출금' 권한은 제외**하세요.
   (본 시스템은 출금 기능을 코드에 아예 구현하지 않습니다.)
3. 키 입력 방법 (둘 중 하나):
   - **대시보드 로그인** — `khali web` 실행 후 화면 상단에서 Key 입력 → 🔑 로그인
     (키는 **메모리에만** 저장, 디스크에 쓰지 않음)
   - **.env 파일** — `BITHUMB_ACCESS_KEY` / `BITHUMB_SECRET_KEY` 입력

## 사용법

```bash
# 1) 백테스트 — 모든 전략 비교
khali backtest --all

# 2) 단일 전략 백테스트
khali backtest

# 3) 웹 대시보드 (기본 명령)
khali web            # http://127.0.0.1:8000

# 4) 헤드리스 매매 루프
khali run

# 5) 실거래 연결 self-check (주문 없이 시세/인증/잔고만 확인)
khali check

# 6) 현재 시장 추세 진단 (지금이 매수 우위 장세인지)
khali regime

# 7) 멀티코인 상대강도 스캔 + BTC 레짐 (무엇을 거래할지)
khali scan
khali scan --markets BTC,ETH,XRP,SOL
```

설치가 막히는 환경에서는 `PYTHONPATH=src python -m khali.main <명령>` 으로도
실행할 수 있습니다.

## 실데이터 백테스트 결과 (XRP/KRW)

빗썸에서 실제 XRP/KRW 캔들을 받아 측정한 결과입니다. **수익률 우선**이라도
하락장에서 자본을 지키는 것이 곧 수익률임을 보여줍니다.

(수수료 **0.04%** 적용)

**최근 ~7개월 (1시간봉 5000개)** — 이 구간 XRP 자체가 **-48.2%** 하락:

| 전략 | 수익률 | MDD | 거래 | 승률 |
|------|:------:|:---:|:---:|:---:|
| 매수후보유 (벤치마크) | **-48.2%** | - | - | - |
| volatility_breakout | **-0.62%** | 0.6% | 6 | 0% |
| rsi_reversion | -2.22% | 4.8% | 12 | 50% |
| ma_crossover | -4.06% | 4.2% | 16 | 25% |

> 시장이 -48% 폭락하는 동안 전략들은 거의 본전을 지켰습니다(현금 보유로 하락 회피).

**전체 일봉 (2017~2026, 약 9년, 강세장 포함)** — 매수후보유 +425%:

| 전략 | 수익률 | MDD | 거래 | 승률 |
|------|:------:|:---:|:---:|:---:|
| rsi_reversion | **+57.5%** | 9.0% | 34 | 47% |
| volatility_breakout | -0.5% | 7.5% | 10 | 20% |
| ma_crossover | -15.6% | 15.6% | 18 | 22% |

직접 재현:
```bash
khali backtest --all --count 5000              # 1시간봉
CANDLE_UNIT=1440 khali backtest --all --count 5000   # 일봉
```

> ⚠️ 과거 성과가 미래 수익을 보장하지 않습니다. 파라미터(`POSITION_SIZE_PCT`,
> 손절·익절 등)와 캔들 주기를 바꿔가며 충분히 검증한 뒤 paper → live 로 진행하세요.

## 파라미터 최적화

`khali optimize` 는 각 전략의 파라미터 조합을 그리드 서치하고, **과최적화를
막기 위해 학습(70%)/검증(30%) 구간을 분리**해 in-sample뿐 아니라
out-of-sample 성과까지 함께 보여줍니다. 기본 점수는 `calmar`(수익률/MDD).

```bash
khali optimize --all                       # 모든 전략 (1h봉)
CANDLE_UNIT=1440 khali optimize --all       # 일봉 기준
khali optimize --strategy rsi_reversion --metric return
```

**실데이터 일봉(9년) 최적화 결과** — 학습 → 검증(미학습 구간) 일반화 성능:

| 전략 | 최적 파라미터 | 학습 수익 | **검증 수익** | 검증 MDD |
|------|------|:---:|:---:|:---:|
| ma_crossover | `short=15, long=100` | +50.5% | **+25.1%** | 3.0% |
| rsi_reversion | `period=14, oversold=30` | +57.5% | **+10.3%** | 8.2% |
| volatility_breakout | `k=0.8` | +113.2% | +9.4% | 3.5% |

> 변동성돌파 `k=0.8` 은 학습 +113%로 화려하지만 검증 +9.4% — 학습 구간에
> 과최적화된 신호입니다. 반면 **`ma_crossover short=15/long=100` 이 검증 구간에서
> +25%로 가장 안정적인 일반화 성능**을 보였습니다. 학습 성과만 보고 고르면 안 됩니다.

찾은 파라미터는 `.env` 의 `STRATEGY_PARAMS`(JSON)에 넣으면 paper/live 매매에
그대로 적용됩니다:
```bash
STRATEGY=ma_crossover
STRATEGY_PARAMS={"short": 15, "long": 100}
```

### 리스크 파라미터 최적화

손절·익절·포지션 비중도 그리드 서치할 수 있습니다 (`--risk`):
```bash
CANDLE_UNIT=1440 khali optimize --strategy ma_crossover --risk
```

### 워크포워드 분석 (가장 엄격한 검증)

`khali walkforward` 는 구간을 굴리며 **매 구간 새로 최적화 → 다음 미학습
구간에서 검증**합니다. 단일 백테스트의 과최적화를 가장 잘 걸러냅니다.

```bash
CANDLE_UNIT=1440 khali walkforward --strategy ma_crossover --train-size 500 --test-size 250
```

**실데이터 일봉(2018~2026) 워크포워드 결과 (ma_crossover):**

| 구성 | 검증 누적(복리) | 수익 구간 비율 |
|------|:---:|:---:|
| 기본 (필터 없음) | **+21.9%** | 64% |
| + 추세필터 MA200 | +1.1% | 36% |

> ⚠️ **중요한 교훈**: 추세필터 MA200 은 *단일* 학습/검증 분할에서는 +57.9%로
> 가장 좋아 보였지만, 워크포워드(여러 구간)에서는 오히려 +1.1%로 부진했습니다.
> 단일 백테스트 수익률은 운에 좌우됩니다 — 그래서 **워크포워드로 교차검증한
> '필터 없는 ma_crossover (+21.9%)' 가 실제로 가장 신뢰할 만한 설정**입니다.
> 코드에 추세필터는 남겨뒀지만(다른 코인/장세용) XRP 기본값은 off 입니다.

## 현시장(XRP) 분석 — 솔직한 결론

`khali regime` 으로 현재 추세를 진단할 수 있습니다. 최근(2026-06) XRP는
가격 < MA50 < MA200 으로 **명확한 하락 추세**입니다.

최근 ~7개월 1h 데이터로 모든 전략을 워크포워드 검증한 결과:

| 시간프레임 | 최선 전략 | OOS 누적 | 비고 |
|------|------|:---:|------|
| 1h (최근 7개월, -48% 하락장) | volatility_breakout | **-7.6%** | 전 전략 마이너스 |
| 6h (3.5년) | rsi_reversion | +2.1% | 거의 본전 |
| 24h (9년, 강세장 포함) | volatility_breakout | +25% | 과거 강세장 덕분 |

> **중요·솔직한 결론**: **지금 같은 하락장에서는 롱(매수) 전용 전략으로 수익을
> 내기 어렵습니다.** 빗썸 현물은 공매도가 안 되므로, 하락장에서 봇이 할 수 있는
> 최선은 *손실 최소화(자본 보존)* 입니다. 실제로 변동성돌파 전략은 시장이 -48%
> 빠지는 동안 -0.8%로 거의 본전을 지켰습니다(대부분 현금 보유).
>
> 즉 **현시장 최적 전략 = 적게 거래하며 기다리기**. `khali regime` 이 🟢(상승)으로
> 바뀌면 그때 자본을 적극 투입하는 것이 수익률에 가장 유리합니다.

**현시장 권장 설정 (자본 보존 우선):**
```bash
CANDLE_UNIT=60
STRATEGY=volatility_breakout
STRATEGY_PARAMS={"k": 0.5}     # k 0.2~0.6 모두 동일하게 방어적, 0.8은 과매매로 손실↑
STOP_LOSS_PCT=0.02
```
추세가 상승 전환(🟢)하기 전까지는 paper 모드로 대기하길 강력히 권합니다.

## 멀티코인 상대강도 + 시장 레짐 (`khali scan`)

전문가 토론에서 합의된 핵심: **'무슨 신호냐'보다 '언제·무엇을 거래하냐'가
수익률을 지배**한다. 단일 코인 고정은 "지는 말에 베팅 고정"이다.

`khali scan` 은 바스켓을 일봉 상대강도(30일 모멘텀 + MA 정렬)로 순위 매기고,
**BTC 레짐**으로 시장 전체가 매수에 우호적인지 판정한다. (알트는 BTC가
약세면 거의 못 오른다.)

- 🟢 BTC 상승 + 1위 코인 상승추세 → 그 코인을 후보로
- 🔴 BTC 하락 → 무엇을 골라도 불리, **현금 대기**가 정답

> 실측(2026-06): 8개 코인 전부 하락, BTC −14%. 상대강도 1위 TRX(−1.9% vsMA50)도
> 시장 약세라 진입 보류 판정. **전 시장 베어장에서는 현금이 최선**임을 도구가
> 그대로 알려준다.

## 멀티코인 로테이션 (`khali rotate`) — 20인 토론 결정 방향

단일 코인 고정을 버리고 **주기적으로 상대강도 1위 코인으로 갈아타되, BTC
레짐이 약세면 현금 보유**하는 전략. 실데이터 백테스트로 검증됨.

```bash
khali rotate                                   # 기본: lookback 120일, 격주 리밸런스(구간검증 robust)
khali rotate --top-n 3                          # 상위 3개 균등분산 (MDD 완화)
khali rotate --lookback 60 --rebalance 14      # 더 민감하게
khali rotate --no-regime                        # BTC 레짐 게이트 끄고 비교
```

**실데이터 검증 (일봉 ~1500일, 8코인):**

| 구성 | 수익률 | MDD | 비고 |
|------|:---:|:---:|------|
| 로테이션(90일/월간) + 레짐 | **+158%** | 57% | BTC(+148%) 상회, 47% 현금 |
| 로테이션(120일/월간) + 레짐 | +333% | 62% | 파라미터 민감 → 과최적화 주의 |
| 매수후보유 XRP | +208% | — | 단일코인 운에 의존 |
| 매수후보유 TRX | +457% | — | 사후적으로만 알 수 있음 |

**최근 ~250일 베어장 (손실 방어 검증):**

| 전략 | 수익률 |
|------|:---:|
| 로테이션 + 레짐게이트 | **-18%** (68% 현금) |
| 매수후보유 BTC | -42% |
| 매수후보유 XRP | -53% |

> 베어장에서 로테이션+레짐은 손실을 **절반 이하**로 줄였습니다(현금 회피). 다만
> 강세장에서는 특정 코인(TRX) 단순보유가 더 나을 수도 있어, 로테이션의 가치는
> **"어느 코인이 오를지 모를 때 분산하며 하락장을 피하는 것"** 입니다.

### 자동 로테이션 라이브 엔진 (`ENGINE=rotation`)

로테이션이 **백테스트를 넘어 실거래 엔진으로 연동**되었습니다. `.env` 에
`ENGINE=rotation` 이면 `khali run`/`khali web` 이 자동으로:
- 매 스텝 **BTC 레짐 확인** → 약세면 즉시 전액 현금 (하락장 회피)
- 강세면 바스켓 중 **상대강도 1위 코인 자동 매수**, 주기적 재선정
- **킬스위치**(`MAX_DRAWDOWN_STOP_PCT`): 고점 대비 설정% 하락 시 전량청산+정지
- 모든 전환에 수수료+슬리피지 반영, 재시작 시 상태 복구, API 레이트리밋

실측(현재): 엔진이 BTC 약세를 읽고 **현금 대기** 판정 — 자본 보존 정상 작동.

## 전략 (플러그인)

| 이름 | 방식 |
|------|------|
| `ma_crossover` | 이동평균 골든/데드크로스 추세추종 |
| `rsi_reversion` | RSI 과매도/과매수 평균회귀 |
| `volatility_breakout` | 변동성 돌파 (래리 윌리엄스, 기본값) |

새 전략은 `src/khali/strategies/` 에 `Strategy` 를 상속하고
`@register("이름")` 만 붙이면 자동 등록됩니다.

## 리스크 파라미터 (`.env`)

| 항목 | 기본값 | 의미 |
|------|:------:|------|
| `POSITION_SIZE_PCT` | 0.5 | 거래당 진입 비중 (자본 대비) |
| `STOP_LOSS_PCT` | 0.02 | 손절선 (-2%) |
| `TAKE_PROFIT_PCT` | 0.04 | 익절선 (+4%) |
| `TRAILING_STOP_PCT` | 0.02 | 트레일링 스탑 (고점 대비 -2%) |
| `DAILY_LOSS_LIMIT_PCT` | 0.1 | 일일 손실 한도 초과 시 당일 중단 |
| `MAX_CONSECUTIVE_LOSSES` | 3 | 연속 손실 N회 후 쿨다운 |
| `MIN_ORDER_KRW` | 5000 | 빗썸 최소 주문 금액 |

## 아키텍처

```
시세(빗썸 API) → 전략 플러그인 → 리스크 매니저 → 주문매니저(모드분기) → DB
                                                          ↘ 웹 대시보드
```

- `exchange/` 빗썸 API 1.0(HMAC) + 2.0(JWT) 클라이언트, 출금 미구현
- `strategies/` 전략 플러그인 + 레지스트리
- `risk/` 손절·익절·사이징·한도 강제 레이어
- `engine/` 포트폴리오·주문매니저·메인 루프
- `backtest/` 과거 데이터 시뮬레이터 + 최적화 + 워크포워드
- `web/` FastAPI 대시보드
- `storage/` SQLAlchemy 거래·자산 기록

## 수익률을 위해 — 데이터로 검증된 권장 설정

여러 검증 끝에 XRP에서 **가장 신뢰할 만한(과최적화 아닌)** 설정:

```bash
CANDLE_UNIT=1440                       # 일봉 (단기 노이즈/수수료 누수 적음)
STRATEGY=ma_crossover
STRATEGY_PARAMS={"short": 15, "long": 100}
TREND_FILTER_MA=0                      # 추세필터는 WFA에서 도움 안 됨 → off
SLIPPAGE_PCT=0.0005                    # 현실적 백테스트용
```
→ 워크포워드(2018~2026) 검증 누적 **+21.9%**, 수익 구간 64%.

검증으로 확인한 **수익률에 실제로 도움되는 것 / 안 되는 것**:
- ✅ **낮은 수수료(0.04%)** — 거래가 잦은 전략일수록 체감 큼
- ✅ **손절·트레일링** — 하락장에서 자본 방어 (1h 하락장 -48% 시장 vs 전략 ~본전)
- ✅ **워크포워드로 고른 파라미터** — 단일 백테스트 1등보다 신뢰도 높음
- ❌ **추세필터 MA200** — 단일 백테스트는 좋아 보였지만 WFA에서 탈락
- ❌ **과한 단타(분봉) + 잦은 거래** — 수수료·슬리피지로 수익 잠식

> 과거 성과 ≠ 미래 수익. 반드시 `khali web` paper 모드로 며칠 검증 후
> `khali check` 로 키 연결을 확인하고, live 는 소액부터 시작하세요.

## 알려진 한계 / 치명적 주의점 (정직한 고지)

코드 감사로 확인·수정한 점과 남은 한계:

- ✅ **수정됨**: 일일 손실 한도가 고정 초기자본 기준 → **현재 평가자산 기준**(복리 반영)
- ✅ **수정됨**: 시뮬레이션 매수가 보유 현금 초과 가능 → 현금 한도로 제한
- ✅ **수정됨**: 재시작 시 포지션 상태 손실 → **DB 영속화·복구**(paper/live)
- ✅ **추가됨**: 로테이션 **라이브 엔진 연동**(`ENGINE=rotation`)
- ✅ **추가됨**: **킬스위치**(최대낙폭 도달 시 청산·정지) + **API 레이트리밋**
- ✅ **추가됨**: **텔레그램 알림**(매수/매도/레짐전환/킬스위치/오류) + 대시보드 자산곡선·레짐 신호등
- ✅ **수정됨**: 라이브 코인선정이 백테스트와 다른 신호(고정30일+MA복합) 사용 →
  백테스트와 동일한 `rotation_lookback` 순수 모멘텀으로 일치 (정합성 결함)
- ℹ️ **검증 결정**: top_n 분산은 4구간 중 2구간에서 MDD를 오히려 높여 *일관된 이점
  없음* → 멀티포지션 라이브 엔진 미구축, **top1 집중 + 레짐게이트**를 robust 기본으로 확정
- ⚠️ **남은 한계**: 대시보드 접근 인증 없음 — `HOST`를 `0.0.0.0`으로 노출 금지
- ⚠️ **남은 한계**: 부분체결·미체결 재시도 미구현 → **live 소액 검증 필수**
- ℹ️ **개별코인 손절**(`ROTATION_STOP_PCT`): 백테스터·엔진 동일 규칙 제공. 단 구간검증상
  레짐게이트와 중복(효과 없음)이라 **기본 off**, 진입가 기준(트레일링 아님)
- ⚠️ **본질적 한계**: 롱 전용 현물이라 하락장 수익 불가, 암호화폐 특성상 MDD가 큼.
  파라미터는 반드시 `walkforward` 로 교차검증 후 사용 (과최적화 = 수익 환상)
